# Research: Lua API for Mode Manager

**Date**: 2026-02-26

## 1. Executive Summary

Implemented Lua API for Mode Manager integration, exposing `jarvis.modes` table with `set_mode()`, `get_current_mode()`, and `get_available_modes()` functions.

## 2. Technical Context

### Existing Lua API Structure
- **Location**: `crates/jarvis-core/src/lua/api/`
- **Registration**: `engine.rs::register_api()`
- **Pattern**: Each API module has `register(lua, jarvis)` function

### Affected Components
- `crates/jarvis-core/src/lua/api/modes.rs` - NEW module
- `crates/jarvis-core/src/lua/api.rs` - Module export
- `crates/jarvis-core/src/lua/engine.rs` - API registration

## 3. Findings & Analysis

### Lua API Functions

#### `jarvis.modes.set_mode(mode_name) -> bool`
```lua
local success = jarvis.modes.set_mode("kid")
if success then
    jarvis.log("info", "Mode switched to kid")
end
```

#### `jarvis.modes.get_current_mode() -> string`
```lua
local current = jarvis.modes.get_current_mode()
jarvis.log("info", "Current mode: " .. current)
```

#### `jarvis.modes.get_available_modes() -> table`
```lua
local modes = jarvis.modes.get_available_modes()
-- Returns: {"normal", "kid", "dev"}
```

### Implementation Details
- Uses global `DB` OnceCell for Settings access
- Creates new `ModesManager` instance per call
- Thread-safe via `parking_lot::RwLock`
- Error handling via Lua runtime errors

## 4. Technical Constraints

- Settings DB must be initialized before Lua scripts run
- Modes API always available (no sandbox restrictions)
- Synchronous API (no async/await)

### Usage Example
```lua
-- Check current mode
local mode = jarvis.modes.get_current_mode()
jarvis.log("info", "Running in " .. mode .. " mode")

-- Switch to kid mode
if jarvis.modes.set_mode("kid") then
    jarvis.log("info", "Switched to kid mode")
    -- Launch kids apps, minimize windows, etc.
end

-- List available modes
local all_modes = jarvis.modes.get_available_modes()
for i, m in ipairs(all_modes) do
    jarvis.log("debug", "Available mode: " .. m)
end
```
