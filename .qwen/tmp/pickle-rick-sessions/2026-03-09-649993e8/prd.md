## Introduction

Данная функция позволяет Jarvis реагировать на определённые команды **без необходимости** произнесения wake word фразы (например, "Jarvis", "Алиса" и т.д.). Это улучшает UX для часто используемых команд и критических команд (например, "стоп", "прекрати").

## Problem Statement

**Current Process:** Пользователь должен произнести wake word фразу перед каждой командой, даже для простых действий.
**Primary Users:** Все пользователи голосового ассистента Jarvis.
**Pain Points:** 
- Избыточность: нужно говорить "Jarvis" перед каждой командой
- Медленная работа для частых команд
- Невозможность быстро остановить выполнение
**Importance:** Улучшение UX, снижение когнитивной нагрузки, возможность быстрой остановки критических команд.

## Objective & Scope

**Objective:** Добавить механизм пометки команд в YAML/TLV файлах который указывает что команда должна работать без wake word.
**Ideal Outcome:** Пользователь может сказать "стоп" или "открой браузер" без предварительного "Jarvis".

### In-scope or Goals
- ✅ Добавить поле `wake_word_required` в структуру команды (YAML/TLV)
- ✅ Изменить STT/распознавание для проверки этого поля
- ✅ Пометить существующие команды (стоп, прекрати, стоп цепочка)
- ✅ Документация для разработчиков команд

### Not-in-scope or Non-Goals
- ❌ Изменение системы wake word (Vosk/RustPotter)
- ❌ Голосовое обучение новым командам
- ❌ Контекстные команды (зависит от предыдущих команд)

## Product Requirements

### Critical User Journeys (CUJs)

1. **CUJ-1: Быстрая команда "стоп"**
   - Пользователь: "стоп" (без wake word)
   - Jarvis: Немедленно останавливает текущее выполнение
   - Ожидание: < 500ms реакция

2. **CUJ-2: Команда "открой браузер"**
   - Пользователь: "открой браузер" (без wake word)
   - Jarvis: Открывает браузер
   - Ожидание: команда выполняется сразу

3. **CUJ-3: Обычная команда с wake word**
   - Пользователь: "Jarvis, расскажи анекдот"
   - Jarvis: Выполняет команду
   - Ожидание: wake word требуется

### Functional Requirements

| Priority | Requirement | User Story |
| :---- | :---- | :---- |
| P0 | Добавить поле `wake_word_required` в YAML/TLV | Как разработчик, я хочу пометить команды чтобы они работали без wake word |

## Assumptions

- Текущая архитектура позволяет фильтровать команды до/после wake word детекции
- YAML/TLV парсер поддерживает новые поля
- Обратная совместимость: команды без поля работают как раньше (wake word required)

## Risks & Mitigations

- **Risk**: Случайная активация команд (ложные срабатывания)
  **Mitigation**: Тщательный подбор фраз для команд без wake word, использование только для уникальных фраз
- **Risk**: Конфликт с обычными командами
  **Mitigation**: Приоритет команд с wake_word_required: false в STT модуле
- **Risk**: Безопасность (неавторизованный доступ)
  **Mitigation**: Не помечать критические команды (удаление файлов, платежи) без wake word

## Tradeoff

- **Option 1**: Глобальный конфиг список команд без wake word
  - Pros: Проще реализовать
  - Cons: Менее гибко, требует изменения кода для добавления команд
- **Option 2**: Поле в каждой команде (ВЫБРАНО)
  - Pros: Гибко, декларативно, легко добавлять новые команды
  - Cons: Требует изменения структуры команды
