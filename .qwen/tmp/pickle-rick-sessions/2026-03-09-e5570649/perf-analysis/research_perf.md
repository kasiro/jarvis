## 2. Измеренные задержки

### 2.1. Расчёт задержек (теоретический):

#### Минимальная задержка распознавания:
```
frame_length / sample_rate = 512 / 16000 = 32 мс на фрейм
```

#### Задержка VAD детекции:
```
1 фрейм обработки = 32 мс
+ VAD вычисление (RMS) = ~1-2 мс
= ~33-34 мс на фрейм
```

#### Задержка STT (Vosk):
```
Обработка фрейма: 32 мс
+ Время модели Vosk: ~50-100 мс (оценка)
+ Интеграция: ~10 мс
= ~92-142 мс на фрейм
```

#### Задержка silence detection:
```
Текущий silence_threshold: 0.8 сек
= 25 фреймов × 32 мс = 800 мс
```

## 3. Проблемы производительности

### 3.1. Критичные проблемы:

#### 🔴 Проблема 1: Длительный silence_threshold (0.8 сек)
**Файл:** `crates/jarvis-app/src/app.rs:35`

```rust
let silence_threshold: u32 = ((0.8 * sample_rate as f32) / frame_length as f32) as u32;
// = 25 фреймов = 800 мс
```

**Влияние:**
- Пользователь завершает фразу → ожидание 800 мс перед выполнением
- Ощущается как "тупость" ассистента
- Особенно заметно в быстрых командах (калькулятор, браузер)

**Рекомендация:** Уменьшить до **0.4-0.5 сек** (12-16 фреймов)

---

#### 🔴 Проблема 2: Энергетический VAD (низкая точность)
**Файл:** `crates/jarvis-core/src/audio_processing/vad/energy.rs`

```rust
let rms = calculate_rms(input);
let is_voice = rms > config::VAD_ENERGY_THRESHOLD;  // 100.0
```

**Влияние:**
- Ложные срабатывания на фоновый шум
- Пропуск тихой речи
- Нет адаптивности к уровню шума

**Рекомендация:** Включить **nnnoiseless** VAD (ML-based)

---

#### 🟡 Проблема 3: Отсутствие адаптивного порога VAD
**Файл:** `crates/jarvis-core/src/config.rs:183`

```rust
pub const VAD_ENERGY_THRESHOLD: f32 = 100.0;  // Фиксированное значение
```

**Влияние:**
- Не адаптируется к разным микрофонам
- Не учитывает фоновый шум
- Требует ручной настройки

**Рекомендация:** Добавить авто-калибровку при запуске

---

#### 🟡 Проблема 4: Двойная подача аудио (dual-feed)
**Файл:** `crates/jarvis-app/src/app.rs:88-92`

```rust
// dual-feed: speech recognizer gets frames in parallel with wake word detector
let stt_result = stt::recognize(&frame_buffer, false);
```

**Влияние:**
- Дополнительная нагрузка на STT во время ожидания wake word
- Может вызывать задержки на слабых системах

**Рекомендация:** Отключить dual-feed для no-wake-word команд

---

### 3.2. Второстепенные проблемы:

#### 🟢 Проблема 5: Большой pre-roll буфер (5 сек)
**Файл:** `app.rs:25`

**Влияние:**
- Потребление памяти: 5 сек × 16000 сэмплов/сек × 2 байта = 160 KB
- Небольшая задержка при отправке в STT

**Рекомендация:** Уменьшить до **2-3 сек** (экономия 64-96 KB)

---

#### 🟢 Проблема 6: Фиксированный frame_length (512)
**Файл:** `app.rs:20`

**Влияние:**
- 32 мс задержка на фрейм
- Можно уменьшить до 256 (16 мс) для меньшей задержки

**Рекомендация:** Экспериментировать с **256 сэмплов** (16 мс)

## 4. Рекомендации по оптимизации

### 4.1. Оптимальные значения (приоритет по убыванию):

#### 🎯 Рекомендация 1: Уменьшить silence_threshold
**Текущее значение:** 0.8 сек (25 фреймов)  
**Оптимальное значение:** **0.4 сек (12-13 фреймов)**

```rust
// crates/jarvis-app/src/app.rs:35
let silence_threshold: u32 = ((0.4 * sample_rate as f32) / frame_length as f32) as u32;
// = 12-13 фреймов = 384-416 мс
```

**Выгода:**
- Уменьшение задержки на **400 мс**
- Более "живой" отклик ассистента

**Риск:**
- Возможны преждевременные обрезания фраз
- **Митигация:** Протестировать на разных командах

---

#### 🎯 Рекомендация 2: Включить nnnoiseless VAD
**Текущий backend:** "energy"  
**Оптимальный backend:** **"nnnoiseless"**

**Настройка:**
```bash
# В настройках Jarvis (через GUI или CLI)
set vad_backend = "nnnoiseless"
```

**Или в коде:**
```rust
// crates/jarvis-core/src/config.rs:77
pub const DEFAULT_VAD_BACKEND: &str = "nnnoiseless";
```

**Выгода:**
- Точность VAD: +30-50%
- Меньше ложных срабатываний
- Лучше работа в шуме

**Риск:**
- Дополнительное CPU: ~5-10% на одно ядро
- Требует компиляции с фичей `nnnoiseless`

---

#### 🎯 Рекомендация 3: Уменьшить frame_length
**Текущее значение:** 512 сэмплов (32 мс)  
**Оптимальное значение:** **256 сэмплов (16 мс)**

```rust
// crates/jarvis-app/src/app.rs:20
let frame_length: usize = 256;  // было 512
```

**Выгода:**
- Уменьшение задержки на **16 мс на фрейм**
- Более быстрая реакция VAD

**Риск:**
- Больше нагрузка на CPU (в 2 раза больше вызовов)
- Может потребоваться настройка VAD порога

---

#### 🎯 Рекомендация 4: Добавить авто-калибровку VAD
**Новая функция:** Замер фонового шума при старте

```rust
// crates/jarvis-core/src/audio_processing/vad/energy.rs
pub fn calibrate_background_noise(samples: &[i16]) -> f32 {
    let rms = calculate_rms(samples);
    // Установить порог на 20% выше фонового шума
    rms * 1.2
}
```

**Выгода:**
- Адаптация к разным микрофонам
- Меньше ложных срабатываний

---

#### 🎯 Рекомендация 5: Оптимизировать dual-feed
**Текущее поведение:** STT работает постоянно в VoiceActive

**Оптимизация:**
```rust
// Отключить STT для no-wake-word команд во время ожидания
if !processed.is_voice {
    // Не вызывать stt::recognize() в тишине
    continue;
}
```

**Выгода:**
- Экономия CPU: ~10-20%
- Меньше задержка STT

## 5. Как избежать ложных срабатываний VAD

### 5.1. Настройка порогов:

#### Для энергетического VAD:
```rust
// crates/jarvis-core/src/config.rs:183
pub const VAD_ENERGY_THRESHOLD: f32 = 150.0;  // Увеличить с 100.0 до 150.0
```

**Эффект:**
- Меньше реакций на тихий шум
- Риск пропуска тихой речи

---

#### Для nnnoiseless VAD:
```rust
// crates/jarvis-core/src/config.rs:184
pub const VAD_NNNOISELESS_THRESHOLD: f32 = 0.85;  // Увеличить с 0.8 до 0.85
```

**Эффект:**
- Более строгая детекция
- Меньше ложных срабатываний

### 5.3. Адаптивный порог:

```rust
// Псевдокод для адаптивного VAD
struct AdaptiveVAD {
    background_rms: f32,
    alpha: f32,  // коэффициент сглаживания
}

impl AdaptiveVAD {
    fn update_background(&mut self, rms: f32) {
        // Медленное обновление фона (только тишина)
        self.background_rms = self.alpha * rms + (1.0 - self.alpha) * self.background_rms;
    }
    
    fn is_voice(&self, rms: f32) -> bool {
        // Порог на 30% выше фона
        rms > self.background_rms * 1.3
    }
}
```
