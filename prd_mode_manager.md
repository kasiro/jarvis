## Introduction

Jarvis имеет существующую Python-реализацию `ModesManager` для управления режимами работы (normal, kid, dev). Однако эта система **НЕ ИНТЕГРИРОВАНА** в ядро команд Rust и работает как отдельный Python модуль. Этот PRD описывает интеграцию менеджера режимов в систему команд Jarvis.

## Problem Statement

**Current Process:** Режимы управляются через отдельные Python скрипты (`kid_mode_on.py`, `kid_mode_off.py`), которые напрямую импортируют `ModesManager`. Нет единого API для команд.

**Primary Users:** 
- Пользователи Jarvis, желающие переключать режимы голосовыми командами
- Разработчики команд, нуждающиеся в доступе к текущему режиму

**Pain Points:**
- Дублирование кода в каждом скрипте режима
- Нет централизованного управления состоянием
- Нет Lua API для доступа к режимам из команд
- Нет Rust API для интеграции с ядром

**Importance:** 
- Упрощение добавления новых режимов
- Централизованное управление состоянием
- Возможность переключения режимов из любых команд

## Objective & Scope

**Objective:** Создать единый Mode Manager API для управления режимами Jarvis с интеграцией в систему команд.

**Ideal Outcome:** 
- Команды могут переключать режимы через Lua API
- Состояние режимов сохраняется между сессиями
- Новые режимы добавляются через конфигурацию

### In-scope or Goals
- Интеграция `ModesManager` в Lua API
- Добавление команд для переключения режимов
- Сохранение состояния режимов в state.db
- Поддержка режимов: normal, kid, dev

### Not-in-scope or Non-Goals
- Изменение существующих Python скриптов режимов
- Добавление новых режимов (кроме существующих)
- Интеграция с Windows (только Linux)

## Product Requirements

### Critical User Journeys (CUJs)

1. **Переключение в детский режим (Kid Mode)**:
   - Пользователь: "запусти детский режим" / "протокол мультики"
   - Jarvis: переключается в kid mode, скрывает окна, запускает YouTube Kids
   - Состояние сохраняется до перезапуска

2. **Выход из детского режима**:
   - Пользователь: "обычный режим" / "выйти из детского режима"
   - Jarvis: переключается в normal mode, восстанавливает окна
   - Закрывает детский браузер

3. **Проверка текущего режима**:
   - Пользователь: "какой режим сейчас?"
   - Jarvis: отвечает текущим режимом

4. **Разработчик проверяет режим в команде**:
   - Разработчик создаёт Lua команду
   - Команда проверяет `jarvis.modes.get_current_mode()`
   - Выполняет логику в зависимости от режима

## Assumptions

- Существующий `ModesManager` в Python будет переписан на Rust или интегрирован через Lua
- State.db уже используется для хранения состояния
- Команды используют YAML/TOML конфигурацию

## Risks & Mitigations

- **Risk**: Конфликт между Python и Rust реализацией -> **Mitigation**: Полная миграция на Rust или единый Lua API
- **Risk**: Потеря состояния при сбое -> **Mitigation**: Атомарная запись в state.db
- **Risk**: Несовместимость с существующими командами -> **Mitigation**: Обратная совместимость через legacy wrapper

## Tradeoff

**Option 1: Полная миграция на Rust**
- Pros: Производительность, типобезопасность, интеграция с ядром
- Cons: Больше работы, нужно переписывать Python код

**Option 2: Lua обёртка над Python**
- Pros: Быстрее, использует существующий код
- Cons: Зависимость от Python, медленнее

**Выбрано: Option 1** - Полная интеграция в Rust для долгосрочной поддержки
