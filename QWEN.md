## Introduction

**Jarvis** — это голосовой ассистент с открытым исходным кодом, разработанный на Rust с использованием Tauri для GUI. Проект представляет собой модульную систему с поддержкой голосовых команд, wake-word детекции, и интеграции с внешними сервисами.

**Контекст**: Проект находится в активной разработке (версия 0.1.0) и использует передовые технологии для распознавания речи (Vosk, RustPotter), выполнения команд через Lua/Python скрипты, и AI fallback через Qwen API.

## Problem Statement

**Current Process**: 
- Пользователь говорит wake-word → Jarvis активируется → распознаёт команду → выполняет действие
- Команды определяются через TOML/YAML конфигурационные файлы
- Выполнение через Python/Lua скрипты или системные команды

**Primary Users**: 
- Конечные пользователи Linux/Windows систем
- Разработчики, расширяющие функциональность через команды

**Pain Points**:
1. **Безопасность**: Команда `unlock` выполняет разблокировку компьютера для ЛЮБОГО человека
2. **Документация**: Фрагментированная документация (QWEN.md, TODO_LIST.md, отсутствующие ADD.md/FIX.md)
3. **Аутентификация**: Отсутствие биометрической или голосовой верификации для критических команд

**Importance**: 
- Безопасность критична для команд типа `unlock`, которые дают доступ к системе
- Надёжная документация необходима для AI ассистентов и разработчиков

## Risks & Mitigations

- **Риск**: Голосовая верификация может быть неточной → **Mitigation**: Использовать комбинацию голоса + пароль
- **Риск**: Биометрические данные могут быть скомпрометированы → **Mitigation**: Хранить хэши, не сырые данные
- **Риск**: Ложные срабатывания wake-word → **Mitigation**: Использовать RustPotter с высоким порогом

## Tradeoff

**Рассмотренные варианты для `unlock`:**

| Вариант | Pros | Cons | Выбор |
| :---- | :---- | :---- | :---- |
| Голосовая верификация | Удобно, естественно | Может быть неточно, нужны ML модели | ⭐ Рекомендуется |
| Пароль + команда | Надёжно | Требует ввода пароля | Альтернатива |
| Комбинация: Голос + Пароль | Максимальная безопасность | Сложнее в реализации | ⭐⭐ Лучший выбор |
| Биометрия (отпечаток) | Очень надёжно | Требует hardware | Не рекомендуется |

## 🎯 КЛЮЧЕВЫЕ КОМПОНЕНТЫ

### 1. Система Команд (commands.rs)
**Расположение**: `crates/jarvis-core/src/commands.rs`

**Архитектура**:
```rust
pub fn parse_commands() -> Result<Vec<JCommandsList>, String>
```

**Процесс**:
1. Чтение директории `resources/commands/`
2. Парсинг `command.toml` или `command.yaml`
3. Конвертация legacy YAML в JCommand
4. Регистрация команд с фразами для распознавания

**Структура команды (TOML)**:
```toml
[[commands]]
action = "python"  # python|lua|cli|ahk
script = "unlock.py"
phrases = ["разблокируй компьютер", "сизам откройся"]
wake_word_required = false
```

### 2. Lua API
**Модуль**: `crates/jarvis-core/src/lua.rs`

**Возможности**:
```lua
jarvis.context.phrase      -- Распознанная фраза
jarvis.context.language    -- "ru" | "en"
jarvis.context.slots       -- Слоты (параметры)
jarvis.state.get("key")    -- Получение состояния
jarvis.state.set("key", val) -- Установка состояния
jarvis.log("info", "msg")  -- Логирование
jarvis.system.notify("Title", "Text") -- Уведомление
jarvis.audio.play_ok()     -- Воспроизвести звук
jarvis.http.get(url)       -- HTTP GET запрос
```

### 3. Wake Word Детекция
**Два режима**:

| Режим | Технология | Флаг | Статус |
| :---- | :---- | :---- | :---- |
| **Vosk** | Vosk STT | (по умолчанию) | ✅ Работает |
| **RustPotter** | RustPotter | `--rustpotter` | ✅ Опционально |

**Сборка с RustPotter**:
```bash
./rebuild.sh --clean --rustpotter
```

### 4. Qwen AI Fallback
**Когда используется**: Когда команда не распознана

**Архитектура**:
1. Пользователь говорит фразу
2. Jarvis не находит匹配ющую команду
3. Отправка текста в Qwen API через FreeQwenApi
4. Qwen возвращает ответ
5. Jarvis озвучивает ответ через TTS

**Прокси**: `FreeQwenApi/` — локальный прокси для Qwen API

## 🔐 БЕЗОПАСНОСТЬ: КОМАНДА `UNLOCK`

### 🚨 ПРОБЛЕМА

**Текущее состояние**:
```python
# resources/commands/windows/unlock.py
async def execute(context):
    jarvis.environment.press_space()
    jarvis.environment.type_text_english(password, char_delay_ms=30, hold_delay_ms=30)
    jarvis.environment.press_enter()
```

**Уязвимость**: **ЛЮБОЙ** может сказать "разблокируй компьютер" и Jarvis выполнит разблокировку!

### ✅ РЕШЕНИЯ (БЕЗ ИЗМЕНЕНИЯ КОДА)

#### Решение 1: Голосовая верификация (Рекомендуется ⭐)

**Архитектура**:
1. Добавить пре-хук для команды `unlock`
2. Перед выполнением запросить голосовую фразу-пароль
3. Сравнить voiceprint с эталоном пользователя
4. Только при совпадении → выполнить разблокировку

**Преимущества**:
- ✅ Удобно (не нужно вводить пароль)
- ✅ Естественно для голосового ассистента
- ✅ Можно комбинировать с wake-word

**Недостатки**:
- ⚠️ Требуется ML модель для voice recognition
- ⚠️ Может быть неточно при шуме

**Реализация** (концептуально):
```python
# Псевдокод
async def execute(context):
    # Шаг 1: Запросить голосовую верификацию
    jarvis.audio.play("verify_voice.wav")
    jarvis.speak("Назовите кодовую фразу для верификации")
    
    # Шаг 2: Распознать голос
    voice_phrase = await jarvis.stt.listen(timeout=10)
    
    # Шаг 3: Проверить voiceprint + фразу
    if not jarvis.auth.verify_voice(voice_phrase, expected_phrase="мой секретный код"):
        jarvis.speak("Голос не распознан. Доступ запрещён.")
        jarvis.log("warning", "Попытка несанкционированной разблокировки!")
        return {"success": False, "error": "voice_mismatch"}
    
    # Шаг 4: Выполнить разблокировку
    jarvis.environment.press_space()
    jarvis.environment.type_text_english(password, char_delay_ms=30, hold_delay_ms=30)
    jarvis.environment.press_enter()
    return {"success": True}
```

#### Решение 2: Двухфакторная аутентификация (Голос + Пароль) ⭐⭐

**Архитектура**:
1. Пользователь говорит "разблокируй компьютер"
2. Jarvis запрашивает пароль (текстом или голосом)
3. Пароль сверяется с хэшем в безопасном хранилище
4. При успехе → разблокировка

**Преимущества**:
- ✅ Максимальная безопасность
- ✅ Два фактора: голос + знание пароля

**Недостатки**:
- ⚠️ Менее удобно (нужно вводить пароль)
- ⚠️ Требует безопасного хранилища паролей

**Реализация** (концептуально):
```python
# Псевдокод
async def execute(context):
    # Шаг 1: Запросить пароль
    jarvis.speak("Введите пароль для разблокировки")
    password_input = await jarvis.input.read_password()
    
    # Шаг 2: Проверить хэш пароля
    stored_hash = jarvis.state.get("unlock_password_hash")
    if not jarvis.auth.verify_hash(password_input, stored_hash):
        jarvis.speak("Неверный пароль")
        jarvis.log("warning", "Неверная попытка пароля")
        return {"success": False, "error": "password_mismatch"}
    
    # Шаг 3: Разблокировка
    jarvis.environment.press_space()
    jarvis.environment.type_text_english(password_input, char_delay_ms=30, hold_delay_ms=30)
    jarvis.environment.press_enter()
    return {"success": True}
```

#### Решение 3: Интеграция с системной аутентификацией

**Архитектура**:
1. Использовать PAM (Linux) или Windows Hello
2. Jarvis вызывает системный API аутентификации
3. Система проверяет отпечаток/лицо/пароль
4. Jarvis получает результат → разблокировка

**Преимущества**:
- ✅ Использует встроенную безопасность ОС
- ✅ Поддержка биометрии

**Недостатки**:
- ⚠️ Требует интеграции с ОС
- ⚠️ Меньше контроля над процессом

#### Решение 4: Временные токены (Одноразовые коды)

**Архитектура**:
1. Пользователь генерирует одноразовый токен
2. Токен действителен N минут
3. Jarvis принимает токен вместо пароля
4. Токен сгорает после использования

**Преимущества**:
- ✅ Защита от replay атак
- ✅ Можно отозвать токен

**Недостатки**:
- ⚠️ Требует управления токенами
- ⚠️ Менее удобно

### 🏆 РЕКОМЕНДАЦИЯ

**Лучшее решение**: **Комбинация Решения 1 + Решения 2**

```
┌─────────────────────────────────────────┐
│  1. Голосовая команда:                  │
│     "разблокируй компьютер"             │
│                                         │
│  2. Голосовая верификация:              │
│     "Назовите кодовую фразу"            │
│     → Проверка voiceprint               │
│                                         │
│  3. Пароль (опционально, если голос     │
│     не распознан):                      │
│     "Введите пароль"                    │
│                                         │
│  4. Разблокировка                       │
└─────────────────────────────────────────┘
```

**Почему это лучше**:
- ✅ Удобство: голосовая верификация по умолчанию
- ✅ Безопасность: пароль как fallback
- ✅ Гибкость: можно настроить порог уверенности голоса
