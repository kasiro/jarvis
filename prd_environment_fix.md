## Introduction

Jarvis - это система голосовой автоматизации на Rust с Lua scripting. Команды `jarvis.environment` управляют запуском внешних приложений (kgx, zeditor, и др.). Текущая реализация имеет критические проблемы со стабильностью, которые делают систему ненадёжной для повседневного использования.

## Problem Statement

**Current Process:** При запуске через Jarvis команды environment работают непредсказуемо - иногда правильно, иногда нет.

**Primary Users:** Пользователи Jarvis, которые используют голосовые команды для запуска приложений и управления рабочими столами.

**Pain Points:**
- kgx запускается с активированным venv проекта jarvis (загрязнение окружения)
- zeditor --reuse запускается долго и на неправильном рабочем столе
- Приложения не запускаются в фоне, блокируют Jarvis
- Рандомное поведение - может запустить всё на одном рабочем столе или нормально
- При выключении Jarvis все запущенные приложения закрываются
- Команды jarvis.environment работают с вероятностью ~50%, нет 100% гарантии

**Importance:** Без стабильности система непригодна для продакшн использования. Пользователи не могут доверять автоматизации.

## Objective & Scope

**Objective:** Достичь 100% стабильности работы команд jarvis.environment.

**Ideal Outcome:** Каждая команда работает предсказуемо, с первого раза, без побочных эффектов.

### In-scope or Goals
- Исправление управления виртуальным окружением (kgx не должен наследовать venv jarvis)
- Исправление запуска zeditor --reuse (быстрый запуск на указанном рабочем столе)
- Запуск приложений в фоне (неблокирующий)
- Детерминированное поведение (100% стабильность)
- Приложения должны оставаться активными после выключения Jarvis
- Стабильная работа всех команд jarvis.environment

### Not-in-scope or Non-Goals
- Добавление новых environment команд
- Изменение функциональности других команд (browser, volume, и т.д.)
- Рефакторинг ядра Jarvis (только environment команды)

## Product Requirements

### Critical User Journeys (CUJs)

1. **Запуск kgx терминала**:
   - Пользователь говорит команду запуска kgx
   - kgx запускается НЕ в venv проекта jarvis
   - kgx запускается на указанном рабочем столе
   - Jarvis сразу разблокирован для следующих команд

2. **Запуск zeditor с reuse**:
   - Пользователь говорит команду запуска zeditor --reuse
   - zeditor запускается быстро (<2 секунд)
   - zeditor открывается на указанном рабочем столе
   - Jarvis не блокируется

3. **Выключение Jarvis**:
   - Пользователь выключает Jarvis
   - Все запущенные приложения (kgx, zeditor, и др.) продолжают работать
   - Jarvis корректно завершает работу

4. **Последовательный запуск команд**:
   - Пользователь запускает несколько команд environment подряд
   - Каждая команда работает на своём рабочем столе
   - Нет рандомного поведения
   - 100% успешных запусков

## Assumptions

- Linux (основная платформа)
- Используется система рабочих столов (GNOME/KDE)
- Приложения запускаются через shell команды
- Jarvis использует Lua scripting для команд

## Risks & Mitigations

- **Risk**: Изменения могут сломать существующие команды → **Mitigation**: Тестирование на Linux перед коммитом, git checkpoint перед изменениями
- **Risk**: Фоновый запуск может привести к утечкам процессов → **Mitigation**: Правильное управление процессами, логирование
- **Risk**: Проблемы с рабочими столами на разных DE → **Mitigation**: Поддержка GNOME (основной), документирование ограничений для других DE

## Tradeoff

- **Option 1**: Полный рефакторинг environment системы
  - Pros: Чистое решение
  - Cons: Долго, высокий риск
  - **Not chosen** - слишком много изменений
  
- **Option 2**: Точечные FIX изменений в существующих командах
  - Pros: Быстро, минимальный риск
  - Cons: Технический долг остаётся
  - **Chosen** - соответствует формату FIX задач
