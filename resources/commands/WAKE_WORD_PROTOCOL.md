# Wake Word Protocol

## Что такое wake_word_required

**`wake_word_required`** - это булево поле в конфигурации команды, которое определяет, требуется ли активация через wake word фразу (например, "Jarvis", "Алиса") для выполнения команды.

### Значения

| Значение | Описание | Пример |
|----------|----------|--------|
| `true` (по умолчанию) | Команда требует wake word | "Jarvis, расскажи анекдот" |
| `false` | Команда выполняется без wake word | "выключись" (без "Jarvis") |

### Механизм работы

1. **STT распознаёт фразу** → пользователь говорит команду
2. **Проверка без wake word** → `fetch_command_no_wake_word()` ищет команды с `wake_word_required: false`
3. **Если найдена** → выполняется немедленно
4. **Если не найдена** → обычный цикл с ожиданием wake word

## ✅ МОЖНО помечать `wake_word_required: false`

### 1. Критические команды безопасности

Команды которые должны работать **всегда**, даже если Jarvis не в активном режиме:

| Команда | Файл | Обоснование |
|---------|------|-------------|
| `terminate` | `terminate/command.yaml` | Экстренное выключение Jarvis |
| `stop_chaining` | (TODO) | Остановка выполнения цепочки команд |

**Пример:**
```yaml
# resources/commands/terminate/command.yaml
list:
- command:
    action: terminate
    wake_word_required: false  # ✅ КРИТИЧНО - должно работать всегда
  phrases:
  - выключись
  - вырубись
  - закройся
  - отключись
  - заверши свою работу
```

### 2. Часто используемые команды

Команды которые пользователь вызывает **много раз в день**:

| Команда | Файл | Обоснование |

```toml
    "включи браузер",
```


## ❌ НЕЛЬЗЯ помечать `wake_word_required: false`

### 1. Voice команды (развлечения)

Команды которые **НЕ должны** выполняться без явного обращения:

| Команда | Файл | Проблема |
|---------|------|----------|
| `пошути` / `расскажи анекдот` | `jarvis/command.yaml` | Может активироваться случайно от фонового разговора |
| `ты дурак` / `ты глупый` | `jarvis/command.yaml` | Реакция на разговоры между людьми |
| `спасибо` / `молодец` | `jarvis/command.yaml` | Реакция на обычную благодарность |
| `пока жди` | `jarvis/command.yaml` | Специфичная команда, не требует частого использования |

**Пример НЕЛЬЗЯ:**
```yaml
# resources/commands/jarvis/command.yaml
# ❌ НЕЛЬЗЯ - voice команды должны требовать wake word!
- command:
    action: voice
    wake_word_required: false  # ❌ ОШИБКА - будет реагировать на фоновые разговоры!
  phrases:
  - расскажи анекдот
  - пошути
  - ты дурак
  - спасибо
```

**Правильно:**
```yaml
# resources/commands/jarvis/command.yaml
# ✅ ПРАВИЛЬНО - voice команды требуют wake word
- command:
    action: voice
    # wake_word_required не указан (по умолчанию true)
  phrases:
  - расскажи анекдот
  - пошути
```

### 2. Опасные системные команды

Команды которые могут **навредить системе** или **нарушить конфиденциальность**:

| Команда | Проблема |
|---------|----------|
| Удаление файлов | Может активироваться случайно → потеря данных |
| Изменение системных настроек | Непреднамеренные изменения конфигурации |
| Блокировка компьютера | "заблокируй компьютер" - может сказаться случайно |
| Скриншоты | "сделай скриншот" - может активироваться от фонового разговора |
| Очистка корзины | Безвозвратное удаление данных |

**Пример НЕЛЬЗЯ:**
```yaml

    action: cli
    cli_cmd: "loginctl lock-session"
    wake_word_required: false  # ❌ ОШИБКА - случайная блокировка!
  phrases:
  - заблокируй компьютер
```

## Примеры нарушений

### ❌ Нарушение 1: Voice команда без wake word

```yaml
# ❌ НАРУШЕНИЕ - voice команда будет активироваться случайно
- command:
    action: voice
    wake_word_required: false  # ❌ ОШИБКА!
  phrases:
  - расскажи анекдот
  - пошути
```

**Проблема:**
- Jarvis будет реагировать на любой разговор про анекдоты
- Фоновый разговор "расскажи мне анекдот" → Jarvis начинает рассказывать
- Ложные срабатывания каждый раз когда кто-то говорит "пошути"

**Исправление:**
```yaml
# ✅ ИСПРАВЛЕНО - удалить wake_word_required или установить true
- command:
    action: voice
    # wake_word_required по умолчанию true
  phrases:
  - расскажи анекдот
  - пошути
```

## Чек-лист проверки

Перед добавлением `wake_word_required: false` проверьте:

### ✅ Разрешающие критерии (должен выполняться ХОТЯ БЫ ОДИН)

- [ ] **Критическая безопасность**: команда должна работать всегда (terminate)
- [ ] **Частое использование**: команда используется >5 раз в день
- [ ] **Уникальные фразы**: фразы которые редко говорятся случайно

### ❌ Запрещающие критерии (НЕ ДОЛЖЕН выполняться НИ ОДИН)

- [ ] **Voice команда**: action = "voice" (развлечения, шутки)
- [ ] **Опасное действие**: удаление, блокировка, скриншоты
- [ ] **Режимы**: переключение режимов работы
- [ ] **Редкое использование**: <1-2 раз в день
- [ ] **Фоновые фразы**: фразы которые могут сказаться в разговоре

### Решение

```
Если (Критическая безопасность ИЛИ Частое использование ИЛИ Уникальные фразы)
И НЕ (Voice команда ИЛИ Опасное действие ИЛИ Режимы ИЛИ Редкое использование ИЛИ Фоновые фразы)
→ МОЖНО установить wake_word_required: false
```
